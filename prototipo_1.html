Prototipo 1

Obtener y abstraer los datos a utilizar



Objetivo

Crear el módulo de conexión de datos apropiado para el manejo de la información de acuerdo a un modelo de datos propuesto diseñado por el equipo de trabajo.

Características

  • El modelo de datos propuesto deberá satisfacer las necesidades para trabajar con diferentes artículos y la interacción con sus usuarios.
  • El sistema deberá permitir la conexión a la fuente de datos que corresponda con el modelo propuesto.
  • El sistema deberá ser capaz de realizar operaciones sobre los datos,para el correcto funcionamiento de los procesos lógicos del sistema.

Restricciones

  • El sistema se verá limitado por el lenguaje de desarrollo Java.
  • Para el caso de estudio, el sistema operará con el gestor de base de datos propio de Neo4j.

Diseño

Problema: Generalización de las características mínimas para realizar recomendaciones.

Solución: Modelo con 3 entidades:

  • Usuarios
  • Articulos
  • Categorías



Resultados del prototipo 1

El sistema tendrá un modelo de datos que parte del modelo general mínimo necesario para trabajar la información añadiendo información relevante sobre el caso de estudio que utiliza platillos como artículos a recomendar.

Las cuales se dividen en:

  • Categoria
  • Platillo
  • Restaurante
  • Usuario
